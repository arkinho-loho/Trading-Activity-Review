# 交易活动回顾 (Trading Activity Review)

## 触发条件

### 触发动词
- `交割单分析` - "分析交割单"、"交割单分析"
- `胜率` - "计算胜率"、"我的胜率"
- `赔率` - "计算赔率"、"赔率分析"
- `凯利仓位` - "凯利公式"、"仓位计算"

### 触发边界
- 包含关键词：`交割单`、`胜率`、`赔率`、`凯利仓位`
- 或使用斜杠命令：`/交割单分析`

### 不触发
- `买`、`卖`、`推荐`、`行情`、`基本面`（交易操作或行情查询）

## 输入输出

### 必需输入
| 字段 | 类型 | 说明 |
|------|------|------|
| file_path | string | 交割单文件路径（.xlsx） |

### 可选输入
| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| include_holdings | boolean | false | 是否纳入持仓计算（暂不启用） |

### 输出
| 字段 | 类型 | 说明 |
|------|------|------|
| excel_path | string | Excel 数据表格路径 |
| report_path | string | Markdown 分析报告路径 |
| error_log | array | 错误日志数组 |
| summary | object | 汇总指标 |

## 处理流程

```
步骤 1: 解析交割单 → 读取Excel + 字段映射 + 数据验证
步骤 2: 证券类型识别 → 代码+名称双重验证
步骤 3: 过滤有效交易 → 仅保留证券买入/卖出
步骤 4: FIFO配对 → 买卖交易配对
步骤 5: 持仓处理 → 合并同一证券代码
步骤 6: 计算指标 → 胜率/赔率/凯利仓位（仅已平仓交易）
步骤 7: 分类统计 → 证券类型 + 持股时长
步骤 8: 生成报告 → Excel + Markdown
```

## 证券类型识别规则

```python
# 代码识别（优先）
# 基金（ETF/LOF）: 50、51、15、56、58、16开头
50/51/15/56/58/16开头 → 基金
11/12开头 → 可转债
60/000/001/002/300/688 → 个股

# 名称识别（备选）
含"ETF" → 基金
含"LOF"/"分级" → 基金
含"转债"/"EB" → 可转债
```

## 核心指标计算

### 胜率
```
胜率 = 盈利笔数 / 总笔数
```

### 赔率
```
赔率 = 平均盈利 / 平均亏损
```

### 凯利仓位
```
凯利 = (胜率 × 赔率 - (1 - 胜率)) / 赔率
     = (p × b - q) / b
其中：p=胜率，b=赔率，q=1-p=败率
```

## Excel输出结构

```
数据表格_YYYY-MM-DD.xlsx
├── 核心指标          # 胜率/赔率/凯利仓位（B列C列均为公式）
├── 持仓明细          # 当前持仓（含Python计算的持有天数）
├── 配对交易          # 已平仓交易（含证券类型、持有期限）
├── 配对交易汇总      # 按证券类型统计
└── 证券类型统计      # 个股/基金/可转债分类统计
```

### 核心指标Sheet
- B列：公式计算值
- C列：公式计算值（与B列一致，便于复制）
- 公式示例：胜率 =B3/B2

### 持仓明细Sheet
- 持有天数：由Python计算（买入日期到当前日期）
- 成本/当前市值/浮动盈亏：使用Excel公式计算

## 分析报告内容

生成的Markdown报告包含：

1. **数据概览** - 交易记录数、时间范围
2. **核心指标** - 胜率、赔率、凯利仓位、总盈亏
3. **综合评估与投资建议**
   - 交易表现评估（胜率/赔率评级）
   - 盈亏分析（盈亏比）
   - 资金管理建议（凯利公式详解）
4. **按证券类型分析** - 个股/基金分类统计
5. **持有期限分析** - 1周/1月/2月/3月/6月以内
6. **当前持仓分析** - 持仓统计、重点持仓、浮动盈亏
7. **风险提示与总结** - 综合建议、风险控制
8. **处理日志** - 错误和警告信息
